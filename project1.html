Aperçu du Projet

Développement complet d'une implémentation du jeu de cartes Pokémon Trading Card Game en Java 21, avec une interface graphique en JavaFX. Le projet s'est déroulé en deux phases distinctes avec des objectifs pédagogiques différents.

Phase 1 : Moteur de jeu

Implémentation de la logique métier du jeu en Java pur :

  • Architecture modulaire : Séparation claire entre les cartes Pokémon, Énergie et Dresseur
  • Système de combat : Gestion des PV, calcul des dégâts, faiblesses et résistances
  • Effets spéciaux : Implémentation des capacités spéciales des Pokémon et des effets de cartes Dresseur
  • Tests unitaires : Couverture de test de 85% avec JUnit pour garantir la fiabilité
  • Qualité de code : Respect des standards Java, documentation Javadoc complète

Phase 2 : Interface graphique

Création d'une interface utilisateur ergonomique avec JavaFX :

  • Design responsive : Adaptation à différentes tailles d'écran
  • Animations : Effets visuels pour les attaques, pioches de cartes et transitions
  • Gestion des événements : Système d'écouteurs pour les interactions utilisateur
  • Écrans multiples : Menu principal, sélection de deck, écran de combat, historique
  • Accessibilité : Support des raccourcis clavier et feedback visuel clair

Défis Techniques

Complexité des règles

Le jeu Pokémon TCG possède des règles complexes avec de nombreuses exceptions. J'ai dû créer un système flexible permettant d'ajouter facilement de nouvelles cartes avec des effets uniques.

Synchronisation État/IHM

Maintenir la synchronisation entre l'état du jeu (backend) et son affichage (frontend) a nécessité l'implémentation du pattern Observer et une gestion fine des événements.

Tests des interactions complexes

Création de tests unitaires pour des scénarios de jeu complexes impliquant plusieurs cartes avec interactions. J'ai développé une bibliothèque de scénarios de test réutilisables.
