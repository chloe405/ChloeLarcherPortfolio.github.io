Infrastructure Serveur Triple Services

Déploiement et sécurisation d'une infrastructure complète avec serveurs MySQL, Apache et Mail

Date : Janvier 2024
Catégorie : Administration Système / Réseaux
Équipe : Projet en binôme
Compétences BUT :
C3 : Administrer C1 : Réaliser C6 : Collaborer

Liens avec les Compétences du BUT

Ce projet d'administration système m'a permis de développer des compétences clés du BUT Informatique, particulièrement pertinentes pour les parcours B et C (Déploiement d'applications et Administration de données).

C3 : Administrer des systèmes informatiques communicants complexes

Niveau 2 atteint : Déployer des services dans une architecture réseau

  • Virtualisation : Création et configuration de machines virtuelles sous VirtualBox
  • Services réseau : Installation et configuration d'Apache, MySQL et Postfix/Dovecot
  • Sécurisation : Configuration de pare-feu (UFW), SSL/TLS et authentification sécurisée
  • Interconnexion : Mise en réseau des VMs et configuration des règles de routage

C1 : Réaliser un développement d'application

Niveau 2 atteint : Partir des exigences et aller jusqu'à une application complète

  • Interface web : Développement d'une interface PHP pour visualiser la base de données
  • Scripting : Création de scripts Bash pour l'automatisation des tâches d'administration
  • Base de données : Modélisation et implémentation d'une base MySQL pour la gestion des mails
  • Tests fonctionnels : Validation du bon fonctionnement de l'ensemble du système

C6 : Collaborer au sein d'une équipe informatique

Niveau 2 atteint : Situer son rôle et ses missions au sein d'une équipe

  • Répartition des tâches : Organisation du travail en binôme avec spécialisation par serveur
  • Documentation partagée : Création d'une documentation technique collaborative
  • Résolution collaborative : Travail en pair programming sur les problèmes complexes
  • Présentation technique : Démonstration conjointe du système fonctionnel

Architecture du Système

Infrastructure à 3 Serveurs

1
Serveur MySQL (VM1)

Base de données relationnelle pour le stockage des utilisateurs et des mails

  • Ubuntu Server 22.04 LTS
  • MySQL 8.0 avec authentification sécurisée
  • phpMyAdmin pour l'administration web
  • Backups automatisés avec mysqldump
2
Serveur Web Apache (VM2)

Interface d'administration et visualisation des données

  • Apache 2.4 avec modules PHP
  • Interface PHP personnalisée
  • Certificat SSL auto-signé
  • Contrôle d'accès par .htaccess
3
Serveur Mail (VM3)

Service complet d'envoi et réception d'emails

  • Postfix pour le transport SMTP
  • Dovecot pour l'accès IMAP/POP3
  • Authentification via MySQL
  • Filtrage anti-spam avec SpamAssassin

Fonctionnement Global

L'architecture permet à deux utilisateurs de s'échanger des emails tout en visualisant l'état de la base de données via une interface web sécurisée :

  1. L'utilisateur se connecte à son client mail (Thunderbird/Outlook)
  2. Le serveur mail authentifie l'utilisateur via la base MySQL
  3. Les emails sont stockés dans la base de données
  4. L'administrateur peut visualiser les statistiques via l'interface web
  5. Toutes les communications sont chiffrées (TLS/SSL)

Défis Techniques

Interconnexion des Services

Faire communiquer trois serveurs distincts sur des machines virtuelles différentes a nécessité une configuration réseau précise (NAT, réseaux internes) et la gestion des règles de pare-feu.

Sécurisation Multi-niveaux

Chaque serveur nécessitait une approche de sécurité spécifique : configuration SSL pour Apache, authentification MySQL sécurisée, et chiffrement des communications mail avec Postfix/Dovecot.

Synchronisation des Authentifications

L'authentification unique entre le serveur mail et la base de données MySQL a nécessité la création d'une structure de données cohérente et des requêtes SQL optimisées.

Solution Implementée

Sécurité

  • Chiffrement : TLS/SSL sur tous les services (HTTPS, SMTPS, IMAPS)
  • Authentification : Système à deux facteurs pour l'administration
  • Pare-feu : Règles UWF restrictives (seuls les ports nécessaires ouverts)
  • Audit : Journalisation centralisée avec logwatch

Performance

  • Optimisation MySQL : Configuration my.cnf adaptée aux ressources limitées des VMs
  • Cache Apache : Activation du cache mod_cache pour l'interface web
  • Monitoring : Scripts de monitoring personnalisés avec alertes

Maintenance

  • Automatisation : Scripts Bash pour les sauvegardes et mises à jour
  • Documentation : Wiki détaillé avec procédures de dépannage
  • Tests : Suite de tests fonctionnels automatisés
